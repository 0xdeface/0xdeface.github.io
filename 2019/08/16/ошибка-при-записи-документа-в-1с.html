Ошибка при записи документа в 1с

16th of August, 2019

  • В 1с ошибка при записи документа ( например чека) или открытия смены возникает в следующих случаях:
  • Идет синхронизация данных которая блокирует таблицы
  • Идет большая транзакция с блокировками
  • Повреждена субд
  • У пользователя не хватает прав на запись или документ не корректен
  1. Порядок решения проблемы (следующий шаг если не помог предыдущий):
    повторить запись документа, возможно транзакция закончилась и он запишется.
  2. перейти в подсистему “НСИ и Администрирование” -> Обслуживание -> Регламентные операции -> Регламентные и фоновые задания. На вкладке фоновые задания отметить только чекбокс выполняется, убедиться что нет блокирующих заданий таких как синхронизация данных ( для риб ). Попробовать записать документ.
  3. Убедить что у пользователя достаточно прав для записи и документ верно оформлен, нет проблем с корректностью данных документа или каких либо еще сообщений, кроме “не удалось записать документ”
  4. [Шаг для файловой базы] Выйти из 1с, сделать бэкап файловой базы, запустить утилиту chdbfl из каталога с 1с, отметить чекбокс исправлять ошибки и запустить проверку. После окончания проверки запустить 1с, в окне запуска для требуемой бд нажать кнопку изменить и указать параметр запуска /ClearCache. Попробовать записать документ. Если документ записан выйти из 1с и убрать указанный ранее параметр запуска.
