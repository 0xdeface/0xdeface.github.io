Настройка EthernetOverUsb

Если вы словили ошибку 235 вашего фн - значит данные от вас в офд не поступали уже месяц, служба EoU либо не запущена, либо не настроена. Для запуска EthernetOverUsb в тестовом режиме для Windows запускайте с параметром -e, для установки в качестве службы с параметром -i. Настройки EoU хранятся в C:\ProgramData\ATOL\EoU

  • В 1с ошибка при записи документа ( например чека) или открытия смены возникает в следующих случаях:

OpenVPN сервер на базе Mikrotik

OPENVPN НА MIKROTIK

Первым делом нам нужно создать шаблоны сертификатов

/certificate
add name=ca-template common-name=example.com days-valid=3650 key-size=2048 key-usage=crl-sign,key-cert-sign
add name=server-template common-name=*.example.com days-valid=3650 key-size=2048 key-usage=digital-signature,key-encipherment,tls-server
add name=client-template common-name=client.example.com days-valid=3650 key-size=2048 key-usage=tls-client

Для OpenVPN сервера поле common name может быть любым. Сертификаты можно сгенерировать используя утилиту easyrsa, но лично у меня они потом не завелись. Клиенты при подключении получали в журнале TCP recv EOF и в логах микротика я видел TLS_ERROR, поэтому буду рад получить обратную связь если у вас заведется со своими сертификатами.

Ошибка при установке/обновлении пакетов в PyCharm

ОШИБКА ПРИ УСТАНОВКЕ (ОБНОВЛЕНИИ) ПАКЕТОВ ЧЕРЕЗ PYCHARM

Собственно делюсь маленьким патчем чтобы старенький PyCharm продолжал исправно работать с новым pip нужно поправить хелпер, путь к нему есть в описании к ошибке. У меня он /pycharm-2018.1/helpers/packaging_tool.py

# pycharm-2018.1/helpers/packaging_tool.py
def do_install(pkgs):
    try:
        import pip
    except ImportError:
        error_no_pip()
    if hasattr(pip,'main'):
        return pip.main(['install'] + pkgs)
    else:
        from pip._internal import main
        return main(['install'] + pkgs)
